# Safety and Responsible-Use Boundaries

**Build 5 · AI Consulting Report Generator · BrightPath ChatGPT Mastery Project**

---

## Absolute Prohibitions

The following must never appear in this system in any phase:

- **Real client records** — client names, live engagement records, commercial records, strategy documents
- **Real learner data** — names, attendance records, progress data, learning outcomes, assessment results
- **Safeguarding case information** — disclosures, referrals, case notes, safeguarding decisions
- **Confidential client records** — commercial information, engagement details, strategic plans
- **Staff HR data** — personnel files, disciplinary records, performance reviews, salary information
- **Personal data** — names, contact details, national insurance numbers, addresses, dates of birth
- **Regulated information** — medical records, financial data, legal case files, insurance records
- **Real organisation data** — actual audit findings, real scores, genuine workflow analysis

All scenarios, organisation profiles, and audit findings must be synthetic and clearly labelled as such.

---

## Data Handling Constraints

- No external AI API calls — all generation is deterministic and local
- No persistent storage of entered data — session state only, cleared on restart
- No cloud deployment in any phase without DPIA, governance, and security review
- No authentication bypass — if authentication is added in future, it must be properly implemented
- No document upload or processing of real client documents
- No production deployment from this prototype codebase without a full privacy, security, and governance review

---

## Output Constraints

Reports generated by this system:

- Are consulting support materials, not certified professional advice
- Do not constitute legal, safeguarding, HR, compliance, financial, medical, or professional advice
- Must be reviewed by a qualified consultant before client delivery
- Must not be presented to clients as a final report without human review and sign-off
- Carry explicit prototype status notices on every page
- Are not suitable for regulated decisions (employment, safeguarding, legal proceedings) without qualified expert review
- Require named responsible owners before any real-world use

---

## Human Review Requirement

Human review is required before:

- Sharing any output with a client, employer, or third party
- Using any output to inform a real AI adoption decision
- Publishing any report or recommendation externally
- Using any risk assessment to justify a compliance position
- Using any training recommendation to design a real training programme

---

## Prototype Status

This is a production-style prototype demonstrating a consulting workflow. It is not:

- A certified consulting or advisory system
- A compliance tool for GDPR, Ofsted, ISO, or any regulatory framework
- A risk management system for production AI deployments
- A substitute for qualified professional advice in any domain
- Production deployed

---

## Governance Before Production Use

Production use of any system like this would require:

- Data Protection Impact Assessment (DPIA)
- Named responsible owner and governance approval
- Security review and penetration testing
- Legal review of output disclaimers
- Safeguarding lead sign-off for any sector with vulnerable people
- HR/employment law review if outputs affect staff
- Regulator notification if applicable (e.g. ICO, sector regulator)
- Pilot testing with synthetic data before live client data
