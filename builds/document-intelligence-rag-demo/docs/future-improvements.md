# Future Improvements — Document Intelligence / RAG Demo

**Build 2 · Prototype v0.6**

These improvements are organised by priority and effort. The build is complete as a deterministic prototype. Everything below is optional enhancement, not missing functionality.

---

## Short-Term Improvements

Low effort, high portfolio or demo value.

- **Capture screenshots** — Use the scenarios in [screenshots-checklist.md](screenshots-checklist.md) to capture the 10 portfolio screenshots. The `assets/screenshots/` folder is already in place.
- **UI polish** — Tighten spacing, add icons to page headings, improve the Evidence-Based Q&A results layout for dense topics
- **PDF export for mini brief** — Use a library like `weasyprint` or a simple `pandoc` call to produce a PDF version of the Markdown brief download
- **Stronger test coverage** — Add edge-case tests for empty documents, very short documents, and documents using unusual formatting
- **Demo presets polish** — Review the suggested questions in the Q&A tab and align them with the most useful demo paths

---

## Medium-Term Improvements

Moderate effort, meaningfully extends capability.

- **Support multiple document sets** — Allow users to switch between different document folders (e.g., "AI policy set" vs "data protection set") without restarting the app
- **Document upload UI** — Allow users to upload their own synthetic or approved Markdown files from the browser, rather than placing them manually in `data/synthetic_documents/`
- **Section-level references** — Return the paragraph context (2–3 lines) around each matched line, not just the matched line itself
- **Improved topic detection** — Weight question keywords by specificity, or add a confidence score per detected topic
- **Semantic search with local embeddings** — Use `sentence-transformers` to embed documents and queries, replace keyword search with cosine similarity retrieval
- **FAISS or Chroma vector store** — Add a local vector store for fast semantic retrieval without an external API
- **Report / CSV export** — Allow evidence results to be downloaded as CSV or JSON for use in spreadsheets or data analysis
- **Markdown preview in Streamlit** — Show a proper rendered Markdown preview of downloaded briefs rather than raw text

---

## Long-Term Improvements

Higher effort, more advanced capability.

- **LLM-assisted Q&A with evidence grounding** — Add an optional LLM answer layer that uses the retrieved evidence snippets as context, with strict source citation and a visible "this answer is generated by AI" disclaimer
- **RAG with embeddings and vector database** — Full retrieval-augmented generation: embed documents, store in FAISS or Chroma, retrieve top-k chunks, generate grounded answers
- **Citation verification** — After generating an LLM answer, verify each claimed source against the actual retrieved evidence to reduce hallucination
- **Document comparison** — Compare two versions of a policy document and identify changes in risk, safeguard, or topic coverage
- **Policy gap analysis** — Automatically identify which of the 13 topics have no policy evidence in the loaded document set
- **Governance checklist generator** — Map detected risks and safeguards to a governance action checklist (e.g., for board reporting)
- **Secure deployment** — Add authentication, role-based access controls, audit logs, and data retention policies for a production deployment
- **Multi-user session isolation** — Support concurrent users without session state collision
- **Document ingestion pipeline** — Add PDF parsing (e.g., `pdfplumber`), Word document parsing, and OCR for scanned documents
- **Operator dashboard** — Show document coverage metrics, topic gaps, and usage analytics across multiple document sets

---

## Not Yet / Avoid For Now

These are out of scope until appropriate governance, technical, and legal reviews are in place:

- **Real client sensitive data** — Do not load real learner records, HR files, safeguarding case documents, or confidential contracts
- **Real safeguarding case documents** — Under no circumstances should safeguarding case notes, referrals, or disclosures be loaded into this prototype
- **HR or disciplinary files** — Staff HR data must not be processed by this tool
- **Production claims** — Do not present this prototype as production-ready or commercially approved
- **Unsupervised legal or compliance interpretation** — All outputs require human review by qualified professionals before action
- **Model-generated answers without evidence and human review** — Any LLM-assisted answer layer must include source citation, limitations, and human review requirements
